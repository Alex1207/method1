Артефакты контейнера: животные: 1.Рыбы (параметр: место проживания – перечислимый тип) 2.Птицы (параметр:отношение к перелету - булевская величина)
Процедурная программа, удовлетворяющая следующим требованиям: 
1.	Запуск программы осуществляется из командной строки, в которой указываются: имя запускаемой программы; имя файла с исходными данными; имя файла с выходными данными.
2.	Для каждого программного объекта, загружаемого в контейнер, исходный файл должен содержать, признак, а также список необходимых параметров. Этот список должен быть представлен в формате, удобном для обработки компьютером. В начале файла указывается размерность контейнера. Сначала вводится ключ, затем признак и имя объекта(см. Подробности.txt).
3.	Функция, формирующая выходной файл, выдает в него программные объекты и их текущие параметры. Помимо этого выводится информацию об общем количестве объектов, содержащихся в контейнере. Информация представлена в форме, удобной для восприятия пользователем.
4.	Разбиение на модули осуществляется по объектному принципу. То есть описание абстракций и функций их обработки группируются по виду абстракций. При этом проводится разделение описания типов, сигнатур и классов от их реализации. Клиентский модуль связан с основной частью программы только минимально необходимым интерфейсом.
5.	Обобщение построено на основе косвенного связывания через универсальный указатель (void* в языке C++).
6.	Контейнер представляет собой хеш массив с разрешением конфликтов через дополнительные списки для элементов с одинаковым значением ключа(суммы кодов символов в имени по модулю 10).
